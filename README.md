# Планировщик задач

Учебный проект (2 курс СПО, ИСП(спо)-209-2, Брянский государственный
инженерно-технологический университет, Многопрофильный колледж) — веб-приложение
«Планировщик задач», реализующее задачу №2 из методички по учебной/производственной
практике.

Совместная работа двух студентов в рамках учебной практики:

- **Магасумов Матвей** — fullstack (фронтенд + бэкенд) по своим функциям, дизайн-макеты для своих экранов в Figma.
- **Шнырёв Илья** — fullstack (фронтенд + бэкенд) по своим функциям, дизайн-макеты для своих экранов в Figma.

## Технологический стек

- **Backend:** Python 3.12, FastAPI, SQLAlchemy
- **БД:** SQLite (файл `tasks.db`)
- **Frontend:** Jinja2-шаблоны, HTML5, CSS3, минимальный JavaScript
- **Аутентификация:** bcrypt + сессии (cookie)
- **Дизайн:** Figma — общий прототип на команду, минимум 2 фрейма на каждого участника
  (есть как ПК-версия, так и мобильная)

по своему набору требований — оценка индивидуальная, поэтому у каждого свои HTML-шаблоны
и свои эндпоинты на FastAPI.

## Возможности приложения

- Регистрация и вход (хеширование паролей через bcrypt, сессионная аутентификация)
- Создание, редактирование и удаление задач
- Приоритеты: низкий / средний / высокий
- Статусы: к выполнению / в работе / выполнено — канбан-доска на главной
- Сроки выполнения с подсветкой просроченных задач и страницей «Уведомления»
- Назначение задач другим пользователям
- Группировка задач по проектам (с произвольным цветом)
- Комментарии к задачам
- Адаптивная вёрстка (десктоп и мобильные устройства)

## Запуск локально

Рекомендуется Python 3.12 (на 3.13 / 3.14 тоже работает, но проверено на 3.12).

### Самый быстрый способ (Windows 10 / 11)

Двойной клик по `run.bat` — он сам:

1. находит Python (`py -3` или `python`),
2. создаёт виртуальное окружение `.venv`,
3. ставит зависимости из `requirements.txt`,
4. заливает демо-данные через `seed_demo.py` (если базы ещё нет),
5. запускает FastAPI на `http://127.0.0.1:8000` и открывает страницу в браузере.

Если что-то пойдёт не так, окно НЕ закроется автоматически — будет сообщение об ошибке
и `pause`, чтобы можно было прочитать причину.

### Через PowerShell вручную (Windows)

```powershell
git clone https://github.com/MatveyHG32/2-PRAKTIKA.git
cd 2-PRAKTIKA
py -3 -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe seed_demo.py
.venv\Scripts\python.exe -m uvicorn main:app --reload
```

### Через bash (Linux / macOS)

```bash
git clone https://github.com/MatveyHG32/2-PRAKTIKA.git
cd 2-PRAKTIKA
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python seed_demo.py
uvicorn main:app --reload
```

Приложение будет доступно на [http://127.0.0.1:8000](http://127.0.0.1:8000).
Документация API (Swagger UI) — на `/docs`.

## Функциональное тестирование (ручной сценарий)

1. Зарегистрировать нового пользователя через `/register`.
2. Создать проект на странице `/projects`.
3. Создать задачу с дедлайном и назначить её другому пользователю.
4. Поменять статус прямо на канбан-доске через выпадающий список.
5. Открыть карточку задачи и добавить комментарий.
6. Открыть страницу «Уведомления» — убедиться, что просроченные / сегодняшние /
   скоро-дедлайн задачи попадают в нужные секции.
7. Открыть страницу в мобильной ширине (DevTools → Responsive) и убедиться, что
   доска перестраивается в одну колонку, а форма редактирования остаётся читаемой.
