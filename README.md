# Планировщик задач

Учебный проект (2 курс СПО) — веб-приложение «Планировщик задач», реализующее задачу №2 из методички
по учебной/производственной практике.

Технологический стек:
- **Backend:** Python 3.12, FastAPI, SQLAlchemy
- **БД:** SQLite (файл `tasks.db`)
- **Frontend:** Jinja2-шаблоны, чистый HTML/CSS, минимум JavaScript
- **Дизайн:** Figma (4 фрейма: 2 для веба, 2 для мобильных устройств)

## Возможности

- Регистрация и вход (хеширование паролей через bcrypt, сессионная аутентификация)
- Создание, редактирование и удаление задач
- Приоритеты (низкий / средний / высокий)
- Статусы (к выполнению / в работе / выполнено) — канбан-доска на главной
- Сроки выполнения с подсветкой просроченных задач
- Назначение задач другим пользователям
- Группировка задач по проектам (с произвольным цветом)
- Комментарии к задачам
- Адаптивная вёрстка (десктоп и мобильные устройства)

## Запуск локально

Рекомендуется Python 3.12 (на 3.13 и 3.14 тоже работает, но проверено на 3.12).

### Самый быстрый способ (Windows)

Двойной клик по `run.bat` — он сам создаст venv, поставит зависимости, зальёт демо-данные
и запустит сервер. Открывай http://127.0.0.1:8000 в браузере.

### Через PowerShell вручную (Windows)

```powershell
git clone -b magasumov https://github.com/MatveyHG32/2-PRAKTIKA.git
cd 2-PRAKTIKA
py -3 -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe seed_demo.py
.venv\Scripts\python.exe -m uvicorn main:app --reload
```

### Через bash (Linux / macOS)

```bash
git clone -b magasumov <repo-url>
cd 2-PRAKTIKA
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python seed_demo.py
uvicorn main:app --reload
```

Приложение доступно на [http://127.0.0.1:8000](http://127.0.0.1:8000).
Документация API (Swagger UI) — на `/docs` (если вам нужно убедиться, что Backend работает).

## Тестирование (функциональное)

Минимальный сценарий ручной проверки:

1. Зарегистрировать нового пользователя через `/register`.
2. Создать проект на странице `/projects`.
3. Создать задачу с дедлайном и назначить её другому пользователю.
4. Поменять статус прямо на канбан-доске через выпадающий список.
5. Открыть карточку задачи и добавить комментарий.
6. Открыть страницу в мобильной ширине окна (или DevTools → Responsive) и убедиться, что
   доска перестраивается в одну колонку, а форма редактирования остаётся читаемой.
